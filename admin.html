SQL Setup
How to run SQL

Go to your Supabase project → SQL Editor → New Query. Paste each block and click Run. Run them in order — Step 1 first, then Step 2, etc.

Step 1 — Update films table (add new columns)
Run this if you already have a films table and need to add the new fields.
ALTER TABLE films ADD COLUMN IF NOT EXISTS director text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS actors text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS country text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS imdb_rating numeric(3,1);
ALTER TABLE films ADD COLUMN IF NOT EXISTS trailer_url text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS content_rating text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS badge text;
ALTER TABLE films ADD COLUMN IF NOT EXISTS status text DEFAULT 'active';
ALTER TABLE films ADD COLUMN IF NOT EXISTS views integer DEFAULT 0;
ALTER TABLE films ADD COLUMN IF NOT EXISTS backdrop_url text;
Step 2 — Create films table from scratch (if you don't have it yet)
Only run this if you haven't created the films table yet.
CREATE TABLE IF NOT EXISTS films (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title text NOT NULL,
  year integer,
  genre text,
  language text,
  country text,
  director text,
  actors text,
  duration_secs integer,
  hls_url text,
  trailer_url text,
  thumbnail_url text,
  backdrop_url text,
  description text,
  imdb_rating numeric(3,1),
  content_rating text,
  badge text,
  featured boolean DEFAULT false,
  status text DEFAULT 'active',
  subtitles jsonb DEFAULT '[]'::jsonb,
  views integer DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

-- Enable RLS
ALTER TABLE films ENABLE ROW LEVEL SECURITY;

-- Public can read active films
CREATE POLICY "Public read active films" ON films
  FOR SELECT USING (status = 'active' OR status IS NULL);

-- Auth users can do everything
CREATE POLICY "Auth full access" ON films
  FOR ALL USING (auth.role() = 'authenticated');
Step 3 — Reactions table (likes / dislikes)
Allows anonymous like/dislike voting using a browser fingerprint.
CREATE TABLE IF NOT EXISTS reactions (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  film_id uuid REFERENCES films(id) ON DELETE CASCADE,
  fingerprint text NOT NULL,
  reaction text CHECK (reaction IN ('like','dislike')),
  created_at timestamptz DEFAULT now(),
  UNIQUE(film_id, fingerprint)
);

ALTER TABLE reactions ENABLE ROW LEVEL SECURITY;

-- Anyone can read reactions (for counts)
CREATE POLICY "Public read reactions" ON reactions
  FOR SELECT USING (true);

-- Anyone can insert/update their own reaction
CREATE POLICY "Public upsert reactions" ON reactions
  FOR INSERT WITH CHECK (true);

CREATE POLICY "Public update reactions" ON reactions
  FOR UPDATE USING (true);

CREATE POLICY "Public delete reactions" ON reactions
  FOR DELETE USING (true);
Step 4 — Banners table (hero carousel)
CREATE TABLE IF NOT EXISTS banners (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  film_id uuid REFERENCES films(id) ON DELETE SET NULL,
  title text,
  subtitle text,
  image_url text,
  cta_text text DEFAULT 'Watch Now',
  cta_url text,
  active boolean DEFAULT true,
  sort_order integer DEFAULT 0,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE banners ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read active banners" ON banners
  FOR SELECT USING (active = true);

CREATE POLICY "Auth manage banners" ON banners
  FOR ALL USING (auth.role() = 'authenticated');
Step 7 — Reviews & Ratings
Run this to enable film reviews and star ratings.
-- Reviews table
CREATE TABLE IF NOT EXISTS reviews (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  film_id uuid REFERENCES films(id) ON DELETE CASCADE,
  fingerprint text NOT NULL,
  author_name text,
  rating integer CHECK (rating BETWEEN 1 AND 5),
  body text NOT NULL CHECK (char_length(body) >= 10),
  helpful_count integer DEFAULT 0,
  created_at timestamptz DEFAULT now(),
  UNIQUE(film_id, fingerprint)
);

ALTER TABLE reviews ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Public read reviews" ON reviews FOR SELECT USING (true);
CREATE POLICY "Auth insert reviews" ON reviews FOR INSERT WITH CHECK (true);

-- Increment helpful count function
CREATE OR REPLACE FUNCTION increment_helpful(review_id uuid)
RETURNS void LANGUAGE sql AS
99 UPDATE reviews SET helpful_count = helpful_count + 1 WHERE id = review_id; 99;
Step 6 — TV Series support
Run this to add series/episode support.
-- Add series columns to films table
ALTER TABLE films ADD COLUMN IF NOT EXISTS type text DEFAULT 'movie';
ALTER TABLE films ADD COLUMN IF NOT EXISTS total_seasons integer DEFAULT 1;

-- Episodes table
CREATE TABLE IF NOT EXISTS episodes (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  series_id uuid REFERENCES films(id) ON DELETE CASCADE,
  season integer NOT NULL DEFAULT 1,
  episode integer NOT NULL DEFAULT 1,
  title text NOT NULL,
  description text,
  hls_url text,
  thumbnail_url text,
  duration_secs integer,
  air_date date,
  created_at timestamptz DEFAULT now()
);

ALTER TABLE episodes ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Public read episodes" ON episodes
  FOR SELECT USING (true);

CREATE POLICY "Auth manage episodes" ON episodes
  FOR ALL USING (auth.role() = 'authenticated');
Step 5 — Views increment function
Creates the RPC function called when a user starts watching a film.
CREATE OR REPLACE FUNCTION increment_views(film_id uuid)
RETURNS void
LANGUAGE plpgsql
SECURITY DEFINER
AS $$
BEGIN
  UPDATE films SET views = COALESCE(views, 0) + 1
  WHERE id = film_id;
END;
$$;

-- Grant public execute
GRANT EXECUTE ON FUNCTION increment_views(uuid) TO anon;
GRANT EXECUTE ON FUNCTION increment_views(uuid) TO authenticated;
Deploy Guide
How to add a new movie

Step 1 — Encode the video

bash encode.sh "/path/to/movie.mp4" film-slug-name

Replace film-slug-name with a URL-friendly name like "the-dark-knight"

Step 2 — Upload to R2

bash upload.sh film-slug-name

Or: bash encode-and-upload.sh movie.mp4 film-slug-name

Step 3 — Add in admin panel

Click "Add Film" → fill in details → paste URL:

https://videos.movxio.com/films/film-slug-name/master.m3u8

Step 4 — For subtitles

wrangler r2 object put movxio-videos/films/film-slug/subtitles/english.vtt \ --file english.vtt --content-type text/vtt
Install FFmpeg (one time)

Windows:

winget install ffmpeg

Mac:

brew install ffmpeg

Verify:

ffmpeg -version
Install Wrangler CLI (one time)
npm install -g wrangler wrangler login wrangler r2 bucket list

How push notifications work

Users who visit MOVXIO and accept the browser notification prompt are subscribed via the PWA install flow. Their subscription data is stored in the push_subscriptions Supabase table. When you click Send Now, this admin panel posts to your Cloudflare Worker which delivers the notification to all subscribers via the Web Push Protocol.

Required: VAPID keys set in your Cloudflare Worker environment variables (VAPID_PUBLIC, VAPID_PRIVATE, VAPID_SUBJECT). See the Deploy Guide for setup instructions.
